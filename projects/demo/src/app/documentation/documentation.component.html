@LocalStored / @SessionStored — Decorator API

Annotate class fields for automatic persistence with deep mutation tracking. Nested property changes and array methods trigger saves automatically.

Deep mutation tracking

The decorator wraps stored objects with proxies via Object.defineProperty. All nested mutations and array method calls trigger automatic saves.

Version management

When the schema changes, increment the version number. Old stored data is discarded and replaced with the new defaults.

Only @LocalStored supports versioning. @SessionStored does not need it.

Signals API — localSignal / sessionSignal

Alternative API that returns a StoredSignal<T> with plain property access and $prop() reactive signals. Must be called in an Angular injection context.

Signal functions

Function Returns Description
localSignal(initial, version, opts) StoredSignal<T> Persisted to localStorage with version invalidation. config.prop for read/write, config.$prop() for reactive signal.
sessionSignal(initial, opts) StoredSignal<T> Persisted to sessionStorage. Same API as localSignal.

Features

  • Decorator API — @LocalStored / @SessionStored with deep mutation tracking on nested objects and arrays
  • Signals API — localSignal() / sessionSignal() returning StoredSignal<T> with plain access and $prop() reactive signals
  • Version invalidation — Increment version to discard stale stored data
  • User-scoped — Prefix storage keys with a user ID via provideStore(userId$)
  • Cross-tab sync — onStorageChange() for localStorage changes from other tabs
  • Error resilient — Handles corrupted JSON and QuotaExceededError gracefully
  • SSR compatible — FakeStorage fallback when window is undefined
  • Zoneless — No dependency on zone.js
